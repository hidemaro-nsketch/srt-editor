# gemini-auto-segmentation Quality Review

Scope: src/routes/api/gemini/analyze.ts, src/routes/index.tsx, src/lib/gemini-client.ts, src/lib/gemini-normalize.ts, src/lib/gemini-schema.ts, src/routeTree.gen.ts.

## Findings

### 1) Gemini normalize endSec can exceed maxDuration
- Severity: High
- File/Line: src/lib/gemini-normalize.ts:41
- Current code:
```ts
const startSec = maxDuration === null ? clampedStart : clamp(clampedStart, 0, maxDuration)
const endSec = maxDuration === null ? clampedEnd : clamp(clampedEnd, 0, maxDuration)
return {
  ...,
  endSec: Math.max(startSec + minGap, endSec),
}
```
- Suggested improvement: After enforcing `minGap`, clamp again or pre-limit `startSec` to `maxDuration - minGap` when duration is known so the final `endSec` never exceeds duration.

### 2) SrtLabelingPage violates single responsibility, too large
- Severity: High
- File/Line: src/routes/index.tsx:36
- Current code:
```ts
function SrtLabelingPage() {
  // media control, timeline editing, keyboard handling,
  // Gemini integration, export, and full rendering
}
```
- Suggested improvement: Split into focused hooks/components (ex: `useMediaPlayer`, `useTimelineDrag`, `useGeminiAnalysis`, `InspectorPanel`, `TimelinePanel`) to reduce function length and improve testability.

### 3) Unsafe double-casts hide real types
- Severity: High
- File/Line: src/routes/index.tsx:379
- Current code:
```ts
const scrollDelta = lane ? lane.scrollLeft - (dragState as unknown as { startScrollLeft: number }).startScrollLeft : 0
```
- Suggested improvement: Include `startScrollLeft` in the drag state type and remove `unknown` casting so the type system validates usage.

### 4) API handler is long and multi-purpose

### 7) Magic numbers in retry/poll/backoff and model config

### 8) Error parsing assumes JSON response
