# SRTラベル付けツール

1. 目的

機械学習モデル作成用に、動画（または音声）に対して時間区間ラベルを付与し、指定フォーマットの .srt を書き出すツールを作る。

2. 出力SRTフォーマット要件（最重要）
2.1 1ブロックの構造

各字幕ブロックは次の3行+空行で構成：

連番（1からの整数、欠番なし）

HH:MM:SS,mmm --> HH:MM:SS,mmm

テキスト行

空行

例：

1
00:00:00,000 --> 00:00:01,000
<b>START</b>

2
00:00:01,000 --> 00:00:01,367
<b>x</b>
2.2 ラベル表記

ラベルは完全一致

余計な空白を入れない

2.3 時刻

ミリ秒3桁固定（,000 など）

start < end を必須（同一は禁止）

内部表現は秒(float)でも良いが、出力は上記に正規化すること

2.4 並び順

開始時刻の昇順で並べる

同一startがある場合は end昇順

並び替え後に連番を振り直す

3. データモデル（内部）
type Segment = {
  id: string
  startSec: number   // seconds, float
  endSec: number     // seconds, float
  label: string      // one of allowed labels
}
4. ラベルセット
4.1 既定ラベル（例）

START

x

END

※実装は labels配列を差し替え可能にする（固定でも良いが、コード内の定数で管理する）

5. 編集フロー（最小機能）
5.1 セグメント作成

ユーザーは以下の手順でセグメントを作れること：

現在時刻を startSec にセット

現在時刻を endSec にセット

label を選ぶ（ドロップダウン or ボタン群）

※「startをセット → endをセット」の順が基本だが、どちらが先でも最終的に整合すること

5.2 セグメント編集

start / end を数値で修正できる

label を変更できる

追加 / 削除できる

5.3 再生時間追従（アクティブ判定）

現在時刻 t が startSec <= t <= endSec のセグメントをアクティブとして視覚的にわかるようにする（ハイライト等）。

6. バリデーション（MVPで必要最低限）

保存（エクスポート）時に以下をチェックし、問題があればエラー表示してエクスポートを止める。

startSec < endSec を満たさない行がある

label が空

時刻がNaN/負数

※セグメントの重なり（オーバーラップ）は MVPでは許容（後で必要なら追加）

7. SRT生成ロジック
7.1 秒 → HH:MM:SS,mmm

秒をミリ秒に変換して整数化（ms = round(sec * 1000) 推奨）

HH は2桁ゼロ埋め、分・秒も2桁、ミリ秒3桁

7.2 SRT本文生成

入力：Segment[]
手順：

startSec昇順、同値ならendSec昇順にソート

1..N の連番を付与

各ブロックのテキストは <b>${label}</b>

ブロック間は空行で区切る

8. エクスポート要件

生成したSRTを .srt としてダウンロードできる

文字コードはUTF-8（BOMは無しでOK）

改行は \n で統一（Windowsでも通常問題なし）

9. 受け入れ基準（この仕様でOKとする条件）

サンプルのように <b>LABEL</b> を含むSRTが出力される

秒編集 → 出力時刻が必ず ,mmm を含む形式に整形される

時刻順に並ぶ & 連番が振り直される

10. 補足（この用途向けの推奨運用）

start は動画先頭に必ず置きたい場合があるので、テンプレとして最初から1行入れてもよい（任意）
